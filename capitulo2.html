Capítulo 2: Instalación y configuración

Requisitos previos

Antes de instalar Stencil.js, asegúrate de tener los siguientes requisitos:

  • 🟢 Node.js v16 o superior
  • 📦 npm o yarn como gestor de paquetes
  • 💻 Un editor de código (recomendado: VS Code)

Instalación del CLI de Stencil

Stencil proporciona un CLI oficial que permite crear y gestionar proyectos fácilmente. Puedes instalarlo globalmente con:


npm init stencil

Este comando descargará e iniciará el generador interactivo para configurar tu proyecto.

Creación de un nuevo proyecto

Una vez ejecutado el comando, se te pedirá elegir un tipo de proyecto:

  • component: para crear una librería de componentes reutilizables.
  • app: para crear una aplicación completa con Stencil.

Por lo general, elegiremos component:


npm init stencil
# Selecciona: component
# Nombre del proyecto: mi-libreria

Estructura básica del proyecto

Una vez creado, verás una estructura como esta:


mi-libreria/
├── stencil.config.ts
├── src/
│   ├── components/
│   │   └── mi-componente/
│   │       ├── mi-componente.tsx
│   │       └── mi-componente.css
├── package.json
├── tsconfig.json
├── index.html

El archivo stencil.config.ts define la configuración del proyecto.

Configuración básica (stencil.config.ts)

Ejemplo de configuración inicial:


// stencil.config.ts
import { Config } from '@stencil/core';

export const config: Config = {
  namespace: 'milibreria',
  outputTargets: [
    {
      type: 'dist',
      esmLoaderPath: 'loader',
    },
    {
      type: 'docs-readme',
    },
    {
      type: 'www',
      serviceWorker: null // desactiva PWA por defecto
    }
  ]
};

Ejecutar el proyecto

Para iniciar el servidor de desarrollo con recarga en caliente:


npm start

Esto abrirá automáticamente el navegador en http://localhost:3333.

🧠 Ejercicio práctico

Crea un nuevo proyecto con npm init stencil. Llámalo mi-biblioteca. Luego:

  • Inspecciona el contenido del archivo stencil.config.ts.
  • Ejecuta el proyecto y observa cómo se renderiza el componente por defecto.
  • Modifica el nombre del componente inicial y cambia el namespace.

💡 Recuerda reiniciar el servidor si haces cambios en la configuración.
